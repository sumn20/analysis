1. UPLOAD-ZONE 上传框 (L565-597)
================================================================================

┌─ 问题描述 ────────────────────────────────────────────────────────────┐
│ 🔴 [高] Padding 不对称                                               │
│         移动/平板: 32px 20px (上下:左右不对称)                       │
│         桌面: 40px 24px                                               │
│         后果: 小屏上内容感觉被挤压                                   │
│                                                                       │
│ 🔴 [高] min-height 偏低                                              │
│         移动端: 140px (过低)                                         │
│         平板+: 160px                                                  │
│         后果: 大屏手机 (6-7") 显示拥挤                               │
│                                                                       │
│ 🟡 [中] 内容溢出处理不足                                            │
│         有 overflow:hidden 但无文本换行控制                         │
│         后果: 长文本可能被裁剪                                       │
└─────────────────────────────────────────────────────────────────────┘

优化前:
  .upload-zone { padding: 32px 20px; min-height: 140px; }
  @media (768px+) { padding: 32px 20px; min-height: 160px; }
  @media (1200px+) { padding: 40px 24px; min-height: 160px; }

优化后:
  .upload-zone { padding: 32px 20px; min-height: 150px; }
  @media (768px+) { padding: 40px 24px; min-height: 160px; }

2. PRIVACY-ALERT 隐私框 (L749-817)
================================================================================

┌─ 问题描述 ────────────────────────────────────────────────────────────┐
│ 🔴 [高] 颜色对比度不足                                               │
│         文本色 #b45309 (浅褐) vs 背景 #fef3c7 (浅黄)                │
│         对比度太低，可能不符合 WCAG 标准                            │
│         后果: 用户难以阅读，可访问性差                             │
│                                                                       │
│ 🟡 [中] 布局在小屏不当折行                                          │
│         inline-flex 容器，标题强制 nowrap                           │
│         后果: 375px 屏幕上标题和文本挤在一起                        │
│                                                                       │
│ 🟡 [中] Padding 不够充足                                            │
│         20px vs footer-info 的 24px                                  │
│         后果: 信息框显示不够宽敞                                     │
│                                                                       │
│ 🟢 [低] 响应式微调不明显                                            │
│         平板只增加 2px (20->20px, 16->18px)                         │
│         后果: 大屏显示不够优雅                                       │
└─────────────────────────────────────────────────────────────────────┘

当前问题:
  - 标题颜色 #92400e (深褐) vs 文本颜色 #b45309 (浅褐)
  - alert-content 使用 flex + align-items:center 导致在小屏折行不当
  - padding 梯度不足

修复方案:
  1. 统一颜色为 #92400e (深褐色)
  2. 改为 flex-direction: column 的列式布局
  3. 增加 padding 梯度: 16px (移动) -> 18px (平板) -> 20px (桌面)

3. RECENT-ANALYSES 最近分析列表 (L2180-2373)
================================================================================

┌─ 问题描述 ────────────────────────────────────────────────────────────┐
│ 🔴 [高] 小屏幕布局混乱，按钮重叠                                      │
│         display: flex + justify-content:space-between 在 375px       │
│         "重新分析" 按钮和文件名可能重叠                             │
│         后果: 用户无法点击，信息显示不完整                          │
│                                                                       │
│ 🔴 [高] 按钮尺寸不一致                                               │
│         "重新分析": padding 6px 12px                                │
│         "删除": padding 6px 8px, min-width: 40px                    │
│         后果: 视觉不协调，删除按钮易误触                           │
│                                                                       │
│ 🟡 [中] item-details 文本溢出未处理                                 │
│         包名可能很长，无 max-width 限制                            │
│         后果: 列表项宽度不稳定                                       │
│                                                                       │
│ 🟡 [中] margin-left 在 space-between 下无效                         │
│         item-actions 的 margin-left: 12px 被忽略                  │
│         后果: 小屏按钮与文本间距不足                               │
└─────────────────────────────────────────────────────────────────────┘

当前布局 (所有屏幕):
  .analysis-item { display: flex; justify-content: space-between; }
  结果: 375px 屏幕上按钮挤在一起

建议布局:
  @media (max-width: 480px) {
    .analysis-item { 
      display: grid; 
      grid-template-columns: 1fr;
      gap: 8px;
    }
    .item-actions { width: 100%; }
    .btn { flex: 1; }
  }
  
  @media (min-width: 481px) {
    .analysis-item {
      display: grid;
      grid-template-columns: 1fr auto;
      gap: 12px;
    }
  }

4. FOOTER-INFO 底部说明区域 (L820-890)
================================================================================

┌─ 问题描述 ────────────────────────────────────────────────────────────┐
│ 🟡 [中] 圆点对齐不准确                                               │
│         圆点 top: 7px (硬编码)                                       │
│         在不同字号下，圆点与文本中线偏离                            │
│         后果: 列表显示不整齐                                         │
│                                                                       │
│ 🟡 [中] 响应式 padding 梯度不足                                      │
│         移动: 20px, 平板: 24px, PC: 24px                           │
│         后果: 梯度感不足，大屏 iPad 显示不优雅                     │
│                                                                       │
│ 🟡 [中] margin-bottom 过渡不当                                       │
│         768px-1023px: 24px                                            │
│         1024px+: 40px                                                │
│         无过渡，差异大                                               │
│         后果: 1000px 和 750px 屏幕 margin 相同不合理               │
│                                                                       │
│ 🟢 [低] 文本溢出处理缺失                                            │
│         超长文本无 word-break 或 overflow-wrap                     │
│         后果: 极端情况下文本可能溢出                               │
└─────────────────────────────────────────────────────────────────────┘

当前代码问题:
  .info-list li::before { top: 7px; } // 硬编码，不够灵活

改进方案:
  .info-list li::before {
    top: 50%;
    transform: translateY(-50%);  // 相对对齐
    margin-top: -0.35em;
  }

5. FILE-UPLOADER 容器规则 (L477-482)
================================================================================

┌─ 问题描述 ────────────────────────────────────────────────────────────┐
│ 🟢 [低] 规则过于宽泛                                                │
│         .file-uploader > :not(.file-uploader-header)                │
│         应用于所有子元素，但子元素本身有更具体的 padding           │
│         后果: 优先级管理混乱，维护困难                             │
│                                                                       │
│ 🟢 [低] 冗余属性                                                    │
│         margin: 0 auto; (元素宽度 100%)                            │
│         max-width: 100%; (与宽度冗余)                             │
│         后果: 代码噪音                                               │
└─────────────────────────────────────────────────────────────────────┘

当前规则:
  .file-uploader > :not(.file-uploader-header) {
    padding: 0 var(--page-padding);
    margin: 0 auto;
    width: 100%;
    max-width: 100%;
  }

建议: 移除此规则或简化为 gap 管理
